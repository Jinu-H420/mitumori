# クラウドコード（Claude Code）制限に引っかかったときの対応

曲げ見積もりの作業中に Claude Code の利用制限で止まったときの選択肢です。

---

## 1. 制限を待つ or プラン変更（クラウドコードをまた使う場合）

- **1. Stop and wait for limit to reset** を選ぶ → リセットまで待ってから再開。
- **2. Switch to extra usage** / **3. Upgrade your plan** → 有料で枠を増やして続行。

---

## 2. クラウドコードなしでできること（今すぐ）

曲げ見積りスキルは、**ターミナルとブラウザだけ**でも次のことができます。

| やりたいこと | やり方 |
|:---|:---|
| 見積りを計算する | **見積り計算書.html** をブラウザで開き、板厚・寸法・材質・形状・数量を入力。計算結果がその場で出る。 |
| evals を実行する | ターミナルで `cd .claude/skills/bending-estimate` のあと `python3 evals/run_evals.py`。成功/失敗が表示される。 |
| Excel 計算書を使う | **曲げ加工見積り_計算書.xlsx** を Excel または Google スプレッドシートで開き、「見積り」シートの入力欄に入力。 |

データ（価格・比重・穴あけ単価）は `data/prices.json`・`data/rates.json`・`data/hole_prices.json` にあり、HTML や evals はここを参照しています。

---

## 3. 続きの開発は Cursor のチャットでやる（おすすめ）

**「evals の期待値更新」や「検索ロジックの統一」など、コードを触る作業**は、**Cursor のこのチャット（私）に依頼**すれば、クラウドコードの制限とは別枠で対応できます。

やってもらいたいことを具体的に書いてください。例:

- 「evals の期待値を新価格表に合わせて更新して」
- 「run_evals.py と estimate_bending.py（または計算ロジック）の検索ロジックを統一して」
- 「縦・横・余長を板幅/展開長にどう対応させるか、計算書の定義に合わせて直して」

必要なファイルの編集・evals の実行・ドキュメントの更新までこちらで行います。

---

## 4. まとめ

| 目的 | 対応 |
|:---|:---|
| クラウドコードをまた使いたい | リミットリセットを待つか、追加利用・プランアップグレード |
| 今すぐ見積りだけ計算したい | 見積り計算書.html か Excel を使う |
| 開発の続き（evals・ロジック統一など）を進めたい | Cursor のチャットで「〇〇して」と依頼する |

制限中でも、**曲げ見積もりの利用と開発の続きはどちらも可能**です。
